# AGENTS.md

本文件说明后续代理在本仓库中协作时应遵守的工作规则。项目是《艾尔登法环》大型 MOD Convergence 的简体中文汉化项目，主要工作是维护 `item` 和 `menu` 目录中的翻译文本。

## 项目结构

- `item/`：道具、武器、防具、地名、NPC 等相关翻译文本。
- `menu/`：菜单、提示、教程、对话事件等相关翻译文本。
- `术语对照表.md`：已经确认并应长期遵守的英文到中文术语表。
- `翻译记忆表.md`：翻译处理记录、保留现状的判断，以及容易误修的反例。
- `README.md`：项目工作流程说明。
- `update/`：游戏更新后提取出的改动文本目录。除非任务明确要求合并更新文本，否则不要主动改动。
- `stage/`：重新打包后的输出目录。

## 翻译文件规则

`item` 和 `menu` 下的 `.fmg.txt` 文件每个条目通常包含以下行：

| 前缀 | 含义 |
| --- | --- |
| `<` | 游戏本体英文文本，仅作参考。 |
| `>` | MOD 英文文本，仅作参考。 |
| `-` | 游戏本体简体中文文本，仅作参考。 |
| `=` | MOD 简体中文译文。翻译和修正时只改这一行。 |

除非用户明确要求，不要修改 `<`、`>`、`-` 行。`=` 行留空表示未翻译，工具处理时会采用英文文本。

## 基本工作流程

1. 先阅读 `README.md`、`术语对照表.md` 和 `翻译记忆表.md`。
2. 使用 `rg` 搜索英文原文、中文译名和相关变体，确认同一英文在不同文件和上下文中的含义。
3. 只在能确认语义和上下文时批量统一。若同一英文在不同系统里表示不同对象，不要机械统一。
4. 手动编辑优先使用 `apply_patch`。
5. 修改后运行残留扫描和 `git diff --check`。
6. 翻译选用优先级:
  * 长单词匹配优先于短词汇。
  * `术语对照表.md` 和 `翻译记忆表.md` 优先级最高
  * 其次是 `*Name.fmg.txt` 里的名词翻译
  * 再次是 `游戏原文对照表.md`
  * 最后是翻译中已知的常用风格

常用检查命令：

```powershell
rg -n "待检查术语|旧译名|可疑译名" item menu 术语对照表.md 翻译记忆表.md 游戏原文对照表.md
git diff --check
git status --short
```

`git diff --check` 中出现 `LF will be replaced by CRLF` 属于当前仓库行尾配置提示，不等同于补丁格式错误。

## 术语处理原则

- 以 `术语对照表.md` 为主，`翻译记忆表.md` 为补充。
- 用户明确确认的译名优先于一般风格判断。
- 与游戏本体官译一致且语义成立的译名，不要为了统一而改写。
- MOD 同名英文可能在不同系统中表示不同对象。必须结合文件、条目类型和上下文判断。
- BOSS 名、追忆名、地名和系统名不要只按字符串相同就统一。
- 润色要克制。设定文本常有官译风格或刻意拗口的表达，不要只因中文更顺就改掉。


## Git 与工作区注意事项

- 工作区可能已有用户修改。不要回滚无关改动。
- 不要删除或清理 `update/`、`stage/`、交换文件等未跟当前任务相关的文件，除非用户明确要求。
- 提交前应查看 `git status --short` 和关键 diff。
- 若只修改翻译文本，不要顺手重排整份文件或做大范围格式化。
